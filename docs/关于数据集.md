## 2. 数据采集

### 2.1 类别定义

本数据集依据 GB 2894 安全标志标准, 选取 5 类常见工业 / 实验室危险警告标志:

| class_id | 目录名 | 中文名 | 说明 |
|:--------:|--------|--------|------|
| 0 | danger | 注意安全 | 感叹号标志, 通用警告 |
| 1 | electricity | 当心触电 | 闪电符号, 电气危险 |
| 2 | fire | 当心火灾 | 火焰符号, 易燃易爆 |
| 3 | ionizing_radiation | 当心电离辐射 | 三叶辐射符号, 放射危险 |
| 4 | train | 当心火车 | 火车符号, 铁路/车辆危险 |

### 2.2 数据来源

| 来源 | 数量 | 比例 | 说明 |
|------|:----:|:----:|------|
| AI 生成 | 435 | 91.0% | 文生图模型生成, 模拟工厂/实验室场景 |
| 网络爬取 | 43 | 9.0% | 真实环境拍摄图片 |
| **合计** | **478** | 100% | |

每张图片只包含**一个**目标 (单个危险标志), 位于图片中心区域附近。

### 2.3 文件命名规则

```
<类别>_<来源>_<序号>.png
```

示例:
- `danger_ai_001.png` — 第 1 张 danger 类 AI 生成图
- `electricity_web_005.png` — 第 5 张 electricity 类网络图

## 3. 数据预处理

### 3.1 流程

```
data/raw/{cat}/*.png
    ↓ prepare_data.py
data/scaled/{cat}/{cat}_{src}_{idx}.jpg    ← 640×640, 去 alpha, 等比例缩放+黑边填充
    ↓ makesense.ai 人工标注
data/labels/{cat}_{src}_{idx}.txt          ← YOLO 格式 bbox
    ↓ split_dataset.py (分层抽样 7:2:1)
data/training/{train|val|test}/{images|labels}/
```

### 3.2 预处理参数

| 参数 | 值 |
|------|-----|
| 目标尺寸 | 640 × 640 |
| 缩放方式 | letterbox (等比例缩放 + 黑边填充) |
| 颜色空间 | BGR (OpenCV 格式), 丢弃 alpha 通道 |
| 输出格式 | JPEG (quality=95) |

### 3.3 数据集划分

| 集合 | 比例 | 数量 |
|------|:----:|:----:|
| train | 70% | 333 |
| val | 20% | 94 |
| test | 10% | 51 |

划分方式: **分层抽样** (每个类别独立按比例划分), 随机种子固定为 42。

## 4. YOLO 标注格式

### 4.1 标注文件格式

每张图片对应一个同名 `.txt` 文件 (放在同目录或 labels 目录下), 每行一个目标:

```
<class_id> <center_x> <center_y> <width> <height>
```

- `class_id`: 整数, 0-4 (见上表)
- `center_x, center_y`: 边界框中心点坐标, **归一化到 [0, 1]** (除以图片宽度/高度)
- `width, height`: 边界框宽高, **归一化到 [0, 1]**

### 4.2 示例

```txt
# fire_ai_001.txt
2 0.512 0.478 0.345 0.367
```

解读: class_id=2 (fire), 边界框中心在 (0.512, 0.478), 宽高为 (0.345, 0.367), (相对于 640×640 图像)。

### 4.3 标注工具

使用 [makesense.ai](https://www.makesense.ai) 在线标注, 选择 YOLO 格式导出。
